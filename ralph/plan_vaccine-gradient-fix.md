# Plan: Fix gradient color interpolation (linear-light blending)

## Context

Gradient fills (e.g., blue‚Üíorange in the vaccine case) appear muddy/brownish in the middle, while Word's reference PDF shows vibrant pink/magenta midtones. This is the classic **gamma-aware blending** problem:

- **Current**: We interpolate gradient stops in sRGB (gamma-encoded) space via `DeviceRGB`. The exponential function does `C0 + t*(C1-C0)` on gamma-encoded values ‚Üí desaturated, dark midtones.
- **Word**: Interpolates in linear-light (gamma-decoded) space ‚Üí vibrant, saturated midtones that preserve perceptual energy.

Evidence from specs:
- OOXML spec defines `gamma` / `invGamma` color transforms for explicit sRGB‚Üîlinear conversion, indicating awareness of linear-light operations
- PDF spec (¬ß8.7.4.4): "The ColorSpace entry determines the colour space in which colour interpolation is performed" and "If ColorSpace is a CIE-based colour space, all gradient fill calculations shall be performed in that space"

## Approach

Use **CalRGB with gamma=1.0** (linear) and the sRGB primaries-to-XYZ matrix. Convert sRGB stop colors to linear RGB before storing in the PDF exponential functions. The PDF viewer then:
1. Evaluates the function ‚Üí linear RGB values (correctly interpolated)
2. Applies CalRGB gamma=1.0 (no-op) + matrix ‚Üí XYZ
3. Converts XYZ to display device ‚Üí proper gamma for screen

This is a one-function change to the gradient rendering code, plus an sRGB-to-linear helper.

## Files to modify

### 1. ‚úÖ `src/pdf/mod.rs` ‚Äî Gradient rendering (lines 1059-1154)

**Add helper function:**
```rust
fn srgb_to_linear(c: u8) -> f32 {
    let s = c as f32 / 255.0;
    if s <= 0.04045 {
        s / 12.92
    } else {
        ((s + 0.055) / 1.055).powf(2.4)
    }
}
```

**Change exponential function color values** (lines 1074-1083, 1096-1105):
- From: `c0[0] as f32 / 255.0`
- To: `srgb_to_linear(c0[0])`

**Change shading color space** (lines 1141-1144):
- From: `.color_space().device_rgb()`
- To: `.color_space().cal_rgb([0.9505, 1.0, 1.0890], None, None, Some([0.4124, 0.2126, 0.0193, 0.3576, 0.715, 0.1192, 0.1805, 0.0722, 0.9505]))`

CalRGB parameters: D65 white point, gamma defaults to [1,1,1] (linear), sRGB primaries matrix.
